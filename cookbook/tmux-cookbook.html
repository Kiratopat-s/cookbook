The tmux cookbook

Every command you'll actually use — sessions, windows, panes, copy mode, config, scripting, and recovery — grouped by practice, with the reasoning behind each one. Copy any line straight into your terminal.

9 practice groups 80+ recipes works with tmux ≥ 3.0 nav lives in the status bar below ↓
!
Two kinds of recipe live here. Shell commands start with $ and run from your normal prompt. Key bindings are pressed inside tmux and always start with the prefix — Ctrl+b by default — released before the key that follows. So "prefix %" means: press Ctrl+b, let go, press %.

Getting started

tmux keeps your terminal work alive on the server side. Close the window, drop the SSH connection, come back tomorrow — your shells, processes, and layout are still there. These are the first five minutes.

Install tmux

# Debian / Ubuntu
$ sudo apt install tmux
# macOS (Homebrew)
$ brew install tmux
# Fedora / RHEL
$ sudo dnf install tmux
# Arch
$ sudo pacman -S tmux

Check what you got with tmux -V. Anything ≥ 3.0 runs every recipe in this book.

Start your first session

$ tmux

Opens a new session with one window and drops you into a shell. The green bar at the bottom is the status line — session name on the left, windows in the middle, clock on the right. (This page's bottom bar imitates it.)

Detach — leave everything running

prefix d
# inside tmux: Ctrl+b, then d
# or as a command from another terminal:
$ tmux detach

Detaching is the whole point of tmux: your programs keep running on the server even after you log out. This is not quitting — it's walking away with the engine on.

Come back (attach)

$ tmux attach
# shorter:
$ tmux a

Re-enters your most recent session exactly as you left it. If you have several sessions, name one with -t — see the Sessions group.

Habit worth building: tmux a || tmux — attach if a session exists, otherwise start one. Many people alias this to t.

The command prompt

prefix :
# Ctrl+b, then :  — a prompt appears in the status line
:new-window -n logs
:kill-pane

Every key binding in tmux is just a shortcut for a command you can type here. If you forget a binding, you can always spell the command out. It has tab-completion.

Built-in help & discoverability

prefix ?
# list every active key binding (q to quit the list)
$ tmux list-keys
# list every command tmux understands
$ tmux list-commands
# the manual is genuinely good:
$ man tmux

prefix+? shows bindings without leaving tmux. list-commands is the fastest way to discover flags you didn't know existed.

The mental model

server ─┬─ session: work ← one project │ ├─ window 0: editor ← like a browser tab │ ├─ window 1: server │ └─ window 2: logs ─┬─ pane 0 ← split of a window │ └─ pane 1 └─ session: personal

One background server holds everything. A session is a workspace (usually one per project). A window is a full-screen tab inside it. A pane is a split inside a window. Every group below maps to one layer of this tree.

1

Sessions — one per project

The habit that makes tmux pay off: a named session for each project, attached and detached at will. Everything here runs from the shell or the command prompt.

Create a named session

$ tmux new -s api
# long form: tmux new-session -s api

Names beat numbers. tmux attach -t api reads like what it does; tmux attach -t 3 does not.

Create it detached (in the background)

$ tmux new -d -s workers
# start it already running a command:
$ tmux new -d -s workers 'npm run worker'

-d creates the session without attaching your terminal to it. This is the building block of every startup script — see the Scripting group.

List sessions

prefix s
$ tmux ls
api: 3 windows (created Fri Jul 3 09:12:44 2026) workers: 1 windows (created Fri Jul 3 09:15:02 2026) (attached)

Inside tmux, prefix+s opens an interactive chooser instead — arrow keys to pick, Enter to jump.

Attach to a specific session

$ tmux attach -t api
# attach and force other clients to detach (fixes wrong-size screens):
$ tmux attach -d -t api

If the session looks shrunk with a border of dots, another (smaller) client is still attached — that's what -d kicks off.

Switch sessions without detaching

prefix ( / )
# from inside tmux (command prompt or a script):
:switch-client -t personal
# prefix (  → previous session,  prefix )  → next session
# prefix L  → last-used session (like alt-tab)

You never need to detach-then-attach to move between projects. prefix+L flipping between your two hottest sessions is a huge quality-of-life win.

Rename a session

prefix $
$ tmux rename-session -t 0 api

Rescue an unnamed session you started with plain tmux. Inside, prefix+$ edits the current one.

Kill a session / kill everything

$ tmux kill-session -t workers
# kill every session EXCEPT the one named api:
$ tmux kill-session -a -t api
# nuke the whole server (all sessions, no undo):
$ tmux kill-server
No undo. Killing a session terminates every process inside it. kill-server takes everything with it.

Check if a session exists (for scripts)

$ tmux has-session -t api 2>/dev/null && echo "running" || echo "not running"

Exit code 0 if the session exists. The standard guard at the top of a project launcher: create the session only if it isn't already there.

Attach-or-create in one line

$ tmux new -A -s api

-A means "attach if it exists, create if it doesn't" — idempotent, perfect for an alias or the last line of an SSH login snippet.

Windows — tabs inside a session

A window fills the whole screen, like a browser tab. Typical setup: window 0 for the editor, 1 for the dev server, 2 for logs, 3 for a scratch shell.

New window

prefix c
:new-window
# with a name, from the shell:
$ tmux new-window -n logs
# open it running a command; window closes when the command exits:
$ tmux new-window -n top 'htop'

New windows open after the current one and get the next free number.

Rename the current window

prefix ,
:rename-window server

Names show in the status bar — 1:server beats 1:zsh. To stop tmux renaming windows after the running program, set set -g allow-rename off in your config.

Move between windows

prefix n / p / 0-9
# prefix n     next window
# prefix p     previous window
# prefix 0..9  jump straight to that number
# prefix l     last-used window (alt-tab)
# prefix '     prompt for a window index
:select-window -t 3

prefix+l (lowercase L) bounces between your two most-used windows — the fastest movement in tmux once it's muscle memory.

Interactive window list

prefix w
:choose-tree -w

A tree of every session and window, with live previews. Arrow keys to move, Enter to jump, x to kill the highlighted item. The best "where was that thing running?" tool tmux has.

Search all windows for text

:find-window 'error'

Matches against window names, titles, and visible content — handy when a stack trace scrolled past in *some* window twenty minutes ago.

Reorder windows

# swap window 3 with window 1:
:swap-window -s 3 -t 1
# move current window to slot 0 (must be free):
:move-window -t 0
# close gaps in numbering (0,1,4,7 → 0,1,2,3):
:move-window -r
Set it and forget it: set -g renumber-windows on in ~/.tmux.conf renumbers automatically whenever a window closes.

Move a window between sessions

# take window 2 of session "api" into session "work":
$ tmux move-window -s api:2 -t work:
# same but keep the original (link, not move):
$ tmux link-window -s api:2 -t work:

Target syntax everywhere in tmux is session:window.pane — e.g. api:2.1 is pane 1 of window 2 in session api.

Kill a window

prefix &
:kill-window
# kill every window except the current one:
:kill-window -a

prefix+& asks for confirmation before killing — the processes inside die with it.

3

Panes — splits you live in

Panes split one window into multiple shells you can see at once: editor on the left, tests on the right, logs on the bottom. This is where tmux keybindings earn their keep.

Split the window

prefix % / "
# prefix %   split left|right (vertical divider)
# prefix "   split top/bottom (horizontal divider)
:split-window -h            # side by side
:split-window -v            # stacked
:split-window -h -p 30      # new pane takes 30% width
:split-window -v 'tail -f app.log'  # split straight into a command
Memory hook: the % character has a vertical-ish slash → vertical divider; " is two marks side by side on a line → horizontal divider. (Or rebind to | and - — see Config.)

Move between panes

prefix ← ↑ ↓ →
# prefix + arrow keys   directional move
# prefix o              cycle to next pane
# prefix ;              last-used pane
:select-pane -L   # -L -R -U -D from scripts

Jump by number

prefix q
:display-panes
# big numbers flash over each pane — press one to jump to it

Faster than arrowing through a 6-pane layout. Extend the flash time with set -g display-panes-time 2000.

Zoom a pane to full screen

prefix z
:resize-pane -Z

Toggles the current pane to fill the window and back, layout untouched. A Z appears next to the window name while zoomed. Probably the single most-loved binding in tmux.

Resize panes

prefix Ctrl+←→↑↓
:resize-pane -L 10   # 10 cells left; also -R -U -D
:resize-pane -x 80   # exactly 80 columns wide
:resize-pane -y 20   # exactly 20 rows tall
Or just drag: with set -g mouse on, grab the divider with the mouse.

Rearrange panes

prefix { / }
# prefix {   swap with previous pane
# prefix }   swap with next pane
:swap-pane -s 0 -t 2   # swap specific panes
:rotate-window          # prefix Ctrl+o — rotate all panes around

Preset layouts

prefix Space
# prefix Space cycles layouts; or jump directly:
# prefix Alt+1  even-horizontal    prefix Alt+2  even-vertical
# prefix Alt+3  main-horizontal    prefix Alt+4  main-vertical
# prefix Alt+5  tiled
:select-layout main-vertical

main-vertical = one big pane on the left, the rest stacked on the right — the classic editor-plus-terminals arrangement.

Break a pane out / merge it back

prefix !
# pane → its own window:
:break-pane
# window 3 → a pane inside the current window:
:join-pane -s 3
# send current pane into window 1 as a split:
:join-pane -t 1

The pair that makes layouts fluid: promote a busy log pane to a full window, pull it back in when things calm down.

Type into every pane at once

:setw synchronize-panes on
# ... run your command on 6 servers at once ...
:setw synchronize-panes off
Sharp tool. Every keystroke goes to every pane in the window — spectacular for fleet SSH work, catastrophic if you forget it's on. Consider a toggle binding: bind S setw synchronize-panes.

Kill a pane

prefix x
:kill-pane
# kill all panes in the window except this one:
:kill-pane -a

Exiting the shell (exit or Ctrl+d) closes a pane too — prefix+x is for when the thing inside is stuck.

Copy mode — scrollback, search, clipboard

tmux keeps its own scrollback per pane. Copy mode freezes the screen so you can scroll, search, and select text with the keyboard — no mouse required.

Enter and leave copy mode

prefix [
# enter: prefix [     leave: q  (or Escape)
:copy-mode

A position indicator like [12/3000] appears top-right: your line offset over total history. With mouse on, scrolling the wheel enters copy mode automatically.

Use vi keys inside copy mode

# ~/.tmux.conf
setw -g mode-keys vi

Turns copy mode into a vim buffer: h j k l to move, w b by word, g / G to history top/bottom, Ctrl+u/d half-page. The recipes below assume vi mode (the default emacs keys are shown where they differ).

Search the scrollback

# in copy mode (vi keys):
#   ?pattern   search UP      /pattern   search DOWN
#   n          next match      N          previous match

Searching up (?) is the common case — the error you want is above you. Matches are highlighted across the whole history.

Select, copy, paste

prefix ]
# vi keys:  Space begins selection → move → Enter copies
#           v (rectangle: Ctrl+v) also begins selection in vi mode
# emacs:    Ctrl+Space begins → Ctrl+w copies
# paste the most recent buffer:
#   prefix ]
:paste-buffer

Copy to the system clipboard

# tmux ≥ 3.2 — usually all you need:
set -g set-clipboard on
# explicit pipe on Linux (X11) if your terminal lacks OSC 52:
bind-key -T copy-mode-vi y send-keys -X copy-pipe-and-cancel "xclip -selection clipboard"
# Wayland: "wl-copy"     macOS: "pbcopy"

Two mechanisms exist: OSC 52 escape codes (terminal does the copying — works over SSH!) and piping to a clipboard tool on the same machine. Try set-clipboard on first; fall back to the pipe.

Work with multiple buffers

prefix =
$ tmux list-buffers
:choose-buffer            # prefix = — pick one interactively to paste
$ tmux show-buffer       # print newest buffer to stdout
$ tmux save-buffer out.txt
$ echo "hello" | tmux load-buffer -   # stdin → paste buffer

Every copy lands in a stack of buffers, so nothing is lost when you copy twice. load-buffer - turns tmux into a paste bridge between shells.

Dump a pane's contents to a file

# visible screen:
$ tmux capture-pane -p > screen.txt
# entire scrollback history:
$ tmux capture-pane -p -S - > full-history.txt
# a specific pane, from outside:
$ tmux capture-pane -p -t api:1.0 -S -100

The forensic tool: grab the crash output of a detached job without ever attaching. -S - means "from the very start of history".

Bigger scrollback

# ~/.tmux.conf — lines kept per pane (default 2000)
set -g history-limit 50000
# wipe the current pane's history:
:clear-history
Applies to new panes only — existing panes keep the limit they were created with. Memory cost is per pane, so 50k is a sane ceiling.

Configuration — ~/.tmux.conf

tmux reads ~/.tmux.conf when the server starts. Everything here is a command you could also type at the prompt — the file is just commands run for you.

Reload config without restarting

:source-file ~/.tmux.conf
# give yourself a binding for it:
bind r source-file ~/.tmux.conf \; display "config reloaded"

The first line to add to any config. Note: source-file runs new commands but can't unset options you removed — for a truly clean slate, tmux kill-server.

Change the prefix (many use Ctrl+a)

unbind C-b
set -g prefix C-a
bind C-a send-prefix

Ctrl+a is closer to the home row (GNU screen refugees already know it). The last line lets a double-tap pass a literal Ctrl+a through to programs that want it.

Turn the mouse on

set -g mouse on

Click to select panes and windows, drag dividers to resize, wheel to scroll into copy mode. Keyboard purism is optional; this one option makes tmux friendly on day one.

Friendlier split bindings

bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"
# make plain "new window" inherit the path too:
bind c new-window -c "#{pane_current_path}"

Two upgrades in one: keys that look like what they do, and new panes that open in the directory you're already in instead of $HOME.

Sane numbering

set -g base-index 1          # windows start at 1
setw -g pane-base-index 1    # panes too
set -g renumber-windows on   # no gaps after closing

Window 1 lives under the 1 key on your keyboard; window 0 lives at the far end. Starting at 1 matches your fingers.

Fix colors and escape lag

set -g default-terminal "tmux-256color"
set -ga terminal-overrides ",*256col*:Tc"   # true 24-bit color
set -s escape-time 10                       # snappy Esc key in vim

Without these: washed-out editor themes and a laggy Esc in vim (tmux waits to see if Esc starts a key sequence). These three lines are in virtually every serious config.

Inspect options — set vs setw, -g and -s

$ tmux show-options -g        # all global session options
$ tmux show-options -g mouse  # one option
$ tmux show-window-options -g

Cheat sheet: set = session options, setw = window options, -g = global default, -s = server-wide. When an option "doesn't work", check you used the right scope.

Style the status bar

set -g status-position top
set -g status-style "bg=#233a2c,fg=#9ce3b0"
set -g status-left " #S "            # session name
set -g status-right " %Y-%m-%d %H:%M "
setw -g window-status-current-style "bg=#35C860,fg=#071309,bold"
set -g status-interval 5              # refresh every 5s

Format strings can embed any tmux variable (#S session, #I window index, #W window name) or shell output via #(command). See them all: man tmux, FORMATS section.

Plugins with tpm

$ git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm

# then at the BOTTOM of ~/.tmux.conf:
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-resurrect'  # save/restore sessions across reboots
set -g @plugin 'tmux-plugins/tmux-continuum'  # …automatically
run '~/.tmux/plugins/tpm/tpm'
# reload config, then prefix I (capital i) to install plugins

resurrect + continuum solve tmux's one real weakness: sessions dying on reboot. prefix+Ctrl+s saves, prefix+Ctrl+r restores.

Scripting & automation

Every tmux action is a CLI command, which means whole workspaces can be built by a shell script. Type your project setup once, run it forever.

send-keys — type into any pane remotely

# run a command in pane 0 of window "server" in session "api":
$ tmux send-keys -t api:server.0 'npm run dev' Enter
# send Ctrl+C to stop it:
$ tmux send-keys -t api:server.0 C-c
# send literal text without interpreting key names:
$ tmux send-keys -l 'echo Enter'

The workhorse of tmux automation. Enter, C-c, Escape, Tab are key names; everything else is typed literally.

Chain commands with \;

$ tmux new -d -s demo \; split-window -h \; select-pane -t 0 \; attach

One tmux invocation can carry many commands separated by \; (escaped so your shell doesn't eat it). Each command acts on the state the previous one left behind.

A complete project launcher

#!/usr/bin/env bash
# dev.sh — build (or rejoin) the "api" workspace
SESSION="api"

if ! tmux has-session -t "$SESSION" 2>/dev/null; then
  tmux new-session  -d -s "$SESSION" -n editor -c ~/code/api
  tmux send-keys    -t "$SESSION:editor" 'nvim .' Enter

  tmux new-window   -t "$SESSION" -n server -c ~/code/api
  tmux send-keys    -t "$SESSION:server" 'npm run dev' Enter
  tmux split-window -t "$SESSION:server" -v -c ~/code/api
  tmux send-keys    -t "$SESSION:server.1" 'npm run test:watch' Enter

  tmux new-window   -t "$SESSION" -n logs -c ~/code/api
  tmux send-keys    -t "$SESSION:logs" 'tail -f log/dev.log' Enter

  tmux select-window -t "$SESSION:editor"
fi

tmux attach -t "$SESSION"

The template to steal: guard with has-session, build detached, attach last. Run it twice and it just reattaches. (For declarative YAML versions of this, look at tmuxinator or tmuxp.)

Query tmux state with formats

$ tmux list-sessions -F '#{session_name}'
$ tmux list-panes -a -F '#{session_name}:#{window_index}.#{pane_index} #{pane_current_command}'
$ tmux display-message -p '#{pane_current_path}'

-F makes list output machine-readable — the glue for fzf pickers and status scripts. display-message -p prints any format for the current context.

Log a pane to a file (pipe-pane)

# start logging everything the pane outputs:
:pipe-pane -o 'cat >> ~/tmux-#S-#W.log'
# run the same command again to stop (it toggles with -o)

Live logging for a deploy or migration you may need to audit later. Unlike capture-pane, this records as it happens, unlimited by scrollback.

Restart whatever a pane was running

:respawn-pane -k
# same for a window:
:respawn-window -k

-k kills the current process first. Handy when a dev server wedges: one command instead of Ctrl+C, up-arrow, Enter.

Conditional config with if-shell

# ~/.tmux.conf — macOS vs Linux clipboard:
if-shell 'uname | grep -q Darwin' \
  'bind -T copy-mode-vi y send -X copy-pipe-and-cancel pbcopy' \
  'bind -T copy-mode-vi y send -X copy-pipe-and-cancel "xclip -sel clip"'

One dotfile repo, many machines. The first string runs if the shell test succeeds, the second if it fails.

Wait for a job to finish (wait-for)

# pane A — run the build, then signal the channel "build-done":
$ make all; tmux wait-for -S build-done
# pane B / script — block until the signal arrives:
$ tmux wait-for build-done && ./deploy.sh

A tiny synchronization primitive between panes and scripts — no polling, no sleep loops.

Popups — a floating terminal (tmux ≥ 3.2)

:display-popup -E -w 80% -h 80% 'lazygit'
# bind a scratch popup to prefix g:
bind g display-popup -E -w 80% -h 80% 'lazygit'

A floating window over your layout — perfect for lazygit, a quick fzf picker, or a throwaway shell. -E closes the popup when the command exits.

Hooks — run commands on events

# announce whenever a session is created:
set-hook -g session-created 'display "new session: #S"'
# auto-rebalance panes whenever one closes:
set-hook -g pane-exited 'select-layout main-vertical'

Event-driven tmux: after-new-window, client-attached, pane-died and dozens more. tmux show-hooks -g lists what's set.

Pro workflows

Patterns that combine everything above: remote work that survives disconnects, pairing, nested sessions, and monitoring.

SSH straight into tmux

$ ssh myserver -t 'tmux new -A -s main'

One command: connect, attach if main exists, create it if not. If your connection drops mid-work, nothing is lost — run the same command and you're back where you were. -t forces the TTY allocation tmux needs.

tmux inside tmux (local + remote)

# send the prefix to the INNER tmux by pressing it twice:
#   Ctrl+b Ctrl+b d   → detaches the inner (remote) session
# or give the inner tmux a different prefix in its ~/.tmux.conf:
set -g prefix C-a

Running local tmux and SSH-ing into a machine that also runs tmux is normal. Prefix-twice targets the inner one; distinct prefixes (outer Ctrl+b, inner Ctrl+a) remove the ambiguity entirely.

Pair programming on one session

# same user account: both simply attach —
$ tmux attach -t pair
# different accounts: share a socket file
$ tmux -S /tmp/pair new -s pair
$ chmod 660 /tmp/pair          # plus a shared group
# partner:
$ tmux -S /tmp/pair attach

Both people see and type into the same panes in real time. If you want to browse different windows of the same session, attach with tmux new -t pair -s mine (a grouped session).

Monitor a window for activity or silence

# highlight window in status bar when output appears:
:setw monitor-activity on
# alert when a window goes QUIET for 30s (build finished):
:setw monitor-silence 30
# make alerts audible/visible:
set -g visual-activity on

Kick off a long build in window 3, go work in window 1, get pinged when it's done. monitor-silence flipping the logic is the underrated half.

Show a big clock

prefix t
:clock-mode

A full-pane clock. Genuinely useful on a spare pane during incident calls; any key dismisses it.

Rename programmatically as you move around

# show the current directory as the window name:
setw -g automatic-rename on
setw -g automatic-rename-format '#{b:pane_current_path}'

#{b:...} is the basename modifier — ~/code/api shows as api. Your status bar becomes a project map for free.

Troubleshooting & rescue

The problems everyone hits, in the order they usually hit them.

"sessions should be nested with care" / can't start tmux

# you're already inside tmux — check:
$ echo $TMUX
# to deliberately nest anyway (rarely wise):
$ TMUX= tmux new -s inner

tmux refuses to start inside itself as a guardrail. Usually the fix is realizing you're already in a session — check the green bar.

Screen framed by dots / stuck small

$ tmux attach -d -t api       # detach the other, smaller client
# or keep both attached but stop sizing to the smallest (tmux ≥ 3.1):
set -g window-size latest

A session renders at the size of its smallest attached client — often a forgotten laptop terminal at home. -d evicts it.

Colors look wrong in vim/nvim

# inside tmux, verify:
$ echo $TERM            # should be tmux-256color or screen-256color
# ~/.tmux.conf:
set -g default-terminal "tmux-256color"
set -ga terminal-overrides ",*256col*:Tc"

Never set TERM yourself inside tmux — set the tmux option and restart the server (tmux kill-server) so it takes effect everywhere.

A pane is frozen

# 1. maybe you hit Ctrl+s (terminal flow control) — unfreeze:
#    press Ctrl+q
# 2. maybe the pane is in copy mode — press q
# 3. still stuck? restart the process in place:
:respawn-pane -k

In that order. Ctrl+s freezes output at the terminal layer and is the classic "tmux is broken" that isn't tmux at all.

Lost? Find every running session and what's in it

$ tmux ls
$ tmux list-panes -a -F '#{session_name}:#{window_index}.#{pane_index}  #{pane_current_command}  #{pane_current_path}'
api:1.0 nvim /home/me/code/api api:2.0 node /home/me/code/api api:2.1 node /home/me/code/api work:1.0 psql /home/me

Config change not taking effect

# reload:
$ tmux source-file ~/.tmux.conf
# find the typo — tmux reports the line number:
$ tmux source-file ~/.tmux.conf 2>&1 | head
# scorched earth (kills everything, reads config fresh):
$ tmux kill-server && tmux

Remember the asymmetry: sourcing applies new lines but can't un-apply deleted ones. Server options and default-terminal in particular need a full server restart.

What is this key bound to?

$ tmux list-keys | grep '\bz\b'
$ tmux list-keys -T copy-mode-vi   # bindings inside copy mode

Key tables (prefix, root, copy-mode-vi…) each hold their own bindings — if a binding "doesn't exist", you may be grepping the wrong table.
